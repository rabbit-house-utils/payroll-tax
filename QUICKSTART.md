# 快速开始指南

## 立即使用

### 1. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:5173 即可使用。

### 2. 输入收入信息

- **月工资**：输入税前月工资（如：30000）
- **年终奖**：输入年终奖金额（如：140000）
- **社平工资**：使用默认值11937或输入当地社平工资

### 3. 配置专项附加扣除（可选）

**方式1：勾选项目**
- ☑ 子女教育：输入人数（每人2000元/月）
- ☑ 住房贷款利息：1000元/月
- ☑ 住房租金：1500元/月（与贷款利息互斥）
- ☑ 赡养老人：3000元/月

**方式2：直接输入总额**
- 勾选"使用自定义扣除额"
- 输入每月扣除总额

### 4. 高级选项（可选）

展开"高级选项"可以配置：
- ☑ 允许年终奖转月工资
- ☑ 允许年终奖转期权
- 期权转换比例（70%-90%）
- 收入类型（期权/股票）
- 期权归属计划

### 5. 查看结果

系统会自动计算并显示三个标签页：

**当前收入明细**
- 年度总收入构成
- 五险一金扣除明细
- 个人所得税明细
- 可支配收入
- 月度收入明细

**四种方案对比**
- 基础方案（不转换）
- 转月工资方案
- 转期权方案
- 双转换方案
- 自动高亮最优方案

**优化建议**
- 推荐最优方案
- 详细调整建议
- 收益分析
- 注意事项

## 典型使用场景

### 场景1：查看当前收入情况

只需输入月工资和年终奖，系统会自动计算：
- 每月到手工资
- 年终奖到手金额
- 全年可支配收入
- 含公积金收入

### 场景2：优化年终奖分配

1. 输入收入信息
2. 勾选"允许年终奖转月工资"
3. 查看"四种方案对比"
4. 系统会自动找出最优转换金额

### 场景3：期权换购决策

1. 输入基础期权金额和归属计划
2. 勾选"允许年终奖转期权"
3. 设置转换比例（如75%）
4. 查看"优化建议"了解是否值得换购

### 场景4：专项附加扣除规划

1. 输入收入信息
2. 勾选可享受的扣除项目
3. 查看税后收入变化
4. 优化扣除项目组合

## 常见问题

### Q: 为什么我的最优方案是"基础方案"？

A: 说明当前收入结构已经比较合理，无需调整。通常发生在：
- 月工资和年终奖比例适中
- 收入总额不高，税率档位较低
- 转换后反而会增加税负

### Q: 转换金额是如何计算的？

A: 系统使用优化算法：
1. 粗搜索：以1000元为步长遍历所有可能
2. 细搜索：在最优解附近以1元为步长精确搜索
3. 找出使可支配收入最大化的转换金额

### Q: 期权转换比例是什么意思？

A: 指年终奖换购期权的折扣率。例如：
- 转换比例75%：用75元年终奖可换购100元期权（相当于25%折扣）
- 转换比例80%：用80元年终奖可换购100元期权（相当于20%折扣）

计算公式：期权价值 = 年终奖金额 / 转换比例

### Q: 计算结果准确吗？

A: 计算基于现行税法，精确到0.01元。但实际情况可能受以下因素影响：
- 地方性税收政策
- 公司具体操作方式
- 其他收入来源
- 建议以实际到手金额为准

### Q: 如何保存我的计算结果？

A: 当前版本暂不支持保存功能。建议：
- 截图保存结果
- 记录关键数据（转换金额、节税金额等）
- 后续版本将支持数据持久化
